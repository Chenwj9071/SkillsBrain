# 查询词模式

## 核心规则

优先使用“动作 + 对象/文件类型”的查询词，能显著提高匹配质量。

示例：

- `提取 PDF 表格`
- `生成 Word 报告`
- `编辑 Excel 公式`
- `JSON 转 CSV`
- `批量重命名文件`

避免只写模糊目标：

- `处理文件`
- `帮我改一下`
- `做个数据操作`

## 不命中时怎么改写

同一个任务第一次没命中时，优先按这三类改写再试：

1. 动作改写：`提取` / `解析` / `导出` / `生成` / `合并`
2. 对象改写：`PDF 表格` / `表格数据` / `图片尺寸` / `日志文件`
3. 场景改写：`批量处理 Excel` / `离线安装依赖` / `增量同步`

## 结果判读建议

- `score >= 0.80`：通常可以直接打开技能并使用
- `0.70 <= score < 0.80`：大概率相关，建议阅读 `SKILL.md` 再确认
- `0.65 <= score < 0.70`：弱相关，建议改写查询词再试
- `< 0.65`：默认不返回（或不建议采用）
