# 查询模式与最佳实践

## 查询关键词策略

### 黄金法则

**任务动词 + 文件类型 = 最有效查询**

| 场景 | 推荐查询 | 避免查询 |
|------|----------|----------|
| PDF表格提取 | `提取PDF表格数据` | `处理PDF` |
| Excel编辑 | `编辑Excel单元格公式` | `改Excel` |
| Word文档 | `创建Word文档模板` | `写文档` |
| 图片处理 | `裁剪图片尺寸` | `改图片` |

---

## 多查询策略

**同一个任务，用不同表述查多次，防止遗漏**

```python
queries = [
    "编辑Excel公式",
    "修改Excel单元格",
    "处理Excel数据"
]

for query in queries:
    results = match_skill(query)
    if results:
        break  # 命中就停
```

## 查询优化技巧

### 1. 包含核心动作词

| ✅ 好 | ❌ 差 |
|-------|-------|
| `提取PDF中的表格` | `PDF表格` |
| `批量修改Excel数据` | `Excel数据` |
| `创建周报文档` | `周报` |

### 2. 指定文件类型

| ✅ 好 | ❌ 差 |
|-------|-------|
| `编辑Excel公式` | `编辑公式` |
| `裁剪JPG图片` | `裁剪图片`（可能命中多个） |

### 3. 添加限定词

| ✅ 好 | ❌ 差 |
|-------|-------|
| `批量重命名文件` | `重命名` |
| `多页PDF表格提取` | `PDF表格` |

## 结果判断标准

### score 解读

| 分数范围 | 含义 |
|----------|------|
| ≥ 0.80 | 高度相关，直接使用 |
| 0.70-0.79 | 相关，建议阅读 SKILL.md 确认 |
| 0.65-0.69 | 弱相关，可能需要调整任务 |
| < 0.65 | 不返回（低于阈值） |

### 命中多个技能时

**优先级判断**：

1. 看 score 分数
2. 看 description 是否完全匹配任务
3. 看 tags 标签
4. 读取 SKILL.md 内容确认

## 常见问题排查

### 1. 返回空数组

**原因**：
- 没有匹配技能
- 查询关键词不匹配
- skill 未启用

**解决**：
- 调整关键词重试
- 检查 `GET /api/skill/list` 看可用技能
- 换一种表述查询

### 2. 返回不相关技能

**原因**：
- 关键词过于模糊
- 多个技能语义重叠

**解决**：
- 添加限定词（文件类型、动作词）
- 用更具体的描述
- 查看 top_k 其他候选

### 3. 未返回期望技能

**原因**：
- 技能未索引
- 阈值过高

**解决**：
- 检查 `GET /api/skill/stats`
- 调用 `POST /api/skill/reindex` 重建索引
- 用更精确的关键词查询

## 集成建议

### Agent 流程

```
用户任务
    ↓
判断任务类型（文件操作/数据处理/自动化）
    ↓
调用 SkillsBrain 查询相关技能
    ↓
有命中？
├── 是 → 读 SKILL.md → 执行技能
└── 否 → 用通用方案执行
```

### 自动查询触发点

建议在 Agent 内部设置自动触发：

```python
TRIGGER_KEYWORDS = [
    # 文件类型
    "PDF", "Excel", "Word", "PPT", "图片", "视频", "音频",
    # 动作词
    "提取", "编辑", "创建", "转换", "批量", "合并", "拆分",
    # 数据类型
    "表格", "数据", "文档", "报告"
]

def should_query_skill(user_input):
    return any(kw in user_input for kw in TRIGGER_KEYWORDS)
```

## 性能建议

1. **启动预热**: Agent 启动时调用一次 `/health` 预热模型
2. **批量查询**: 同一任务多个查询，可以并行调用
3. **缓存会话**: 同一会话的查询结果可短期缓存
4. **超时设置**: 建议 timeout=10s，服务响应通常 <50ms
